ui-sortable : 排序

介绍

使用这个插件可以用鼠标拖动list中的元素进行排序

版本

  • 倪朱国: 创建----2011.10.20

开始

<ul id="sortable">
    <li>item 2</li>
    <li>item 3</li>
    <li>item 4</li>
</ul>
jQuery(function($){
    $.use("ui-sortable",function(){
        $("#sortable").sortable(); 
    });
});

Configs

disabled : Boolean
Default: false
使用(false)或禁用(true) sortable，可以在初始化sortable的时候设置。
appendTo : String
Default: "parent"
指定helper追加在哪个元素后面
axis : String
Default: false
如果被定义了，则选项只能被水平或者垂直托动,可选的值："x","y"
cancel : Selector
Default: ":input,button"
阻止排序动作在匹配的元素上发生
connectWith : Selector
Default: false
允许sortable对象连接另一个sortable对象，可将item元素拖拽到另一个中。
containment : Element, String, Selector
Default: false
约束排序动作只能在一个指定范围内发生。
注意：给定的元素必须要有确切的宽和高。
cursor : String
Default: "auto"
指定在排序时光标的显示样式
cursorAt : Object
Default: false
排序时光标相对元素所在的位置，值为{ top, left, right, bottom}中的一 个或两个。
delay : Integer
Default: 0
以毫秒为单位，设置延迟多久才激活排序动作。此参数可防止误点击。
distance : Integer
Default: 1
决定至少要在元素上面拖动多少像素后，才正式触发排序动作。
dropOnEmpty : Boolean
Default: true
是否允许拖拽到一个空的sortable对象中。
forceHelperSize : Boolean
Default: false
If true, forces the helper to have a size.
forcePlaceholderSize : Boolean
Default: false
If true, forces the placeholder to have a size.
grid : Array
Default: false
将排序对象的item元素视为一个格子处理，每次移动都按一个格子大小移动，数组值：[x, y]
handle : Selector, Element
Default: false
限制排序的动作只能在item元素的某个元素开始。
helper : String, Function
Default: "original"
设置是否在拖拽元素时，显示一个辅助的元素。可选值： 'original','clone'
items : Selector
Default: "> *"
指定在排序对象中，哪些元素可以进行拖拽排序的。
opacity : Float
Default: false
指定helper的透明度。从 0.01 到 1
placeholder : String
Default: false
设置当排序动作发生时，空白占位符的CSS样式。
revert : Boolean/Integer
Default: false
如果设置为true，则被拖拽的元素在返回新位置时，会有一个动画效果。
参数也可以是一个数字，指定动画的时间。
scroll : Boolean
Default: true
如果为true，则元素被拖动到页面边缘时，会自动滚动。
scrollSensitivity : Integer
Default: 20
设置当元素移动至边缘多少像素时，便开始滚动页面。
scrollSpeed : Integer
Default: 20
设置页面的滚动速度。
tolerance : String
Default: "intersect"
设置当拖动元素越过其他元素多少时便对元素进行重新排序。有两个可选值'intersect','pointer'.
intersect: 和另一个元素重叠部分超过50%
pointer: 光标和两一个元素重叠
zIndex : integer
Default: 1000
设置在排序动作发生时，元素的z-index值。

Methods

.sortable("destroy")
彻底删除sortable。这将使元素返回到初始状态
.sortable("disable")
使sortable失效
.sortable("enable")
使sortable生效
.sortable("option",optionName,[value])
  • optionName : String 参数名
获得或更改sortable的参数 
//getter
var optionName = $("...").sortable("option",optionName);
//setter
$("...").sortable("option",optionName,value);
.sortable("option", options)
  • options : Object 参数对象
提供了一个选项object，一次设置多个sortable元素。
.sortable("widget")
返回 .ui-sortable 元素
.sortable("serialize", [options])
获取或设置序列化后每个item元素的id属性。
.sortable("toArray")
获取序列化后的每个item元素的id属性的数组。
.sortable("refresh")
手动刷新当前sortable对象的item元素排序。
.sortable("refreshPositions")
手动重新刷新当前sortable对象的item元素的坐标，此方法可能会降低性能。
.sortable("cancel")
取消当前sortable对象中item元素的排序改变。

Events

create Type: sortcreate

sortable全部事件的参数都如下，其他事件的参数也请参考这里的。

  • event : Event
  • ui : Obecjt 
    {
    helper:         // 表示sortable元素的jQuery对象，通常是当前元素的克隆对象
    position:       // 当前helper元素的位置
    offset:         // 当前helper元素的绝对位置
    item:           // 当前拖动的元素
    placeholder:    // 占位符(如果有的话)
    sender:         // 当前拖动的元素从哪个list而来。(只有在多个list之间拖动时才有用)
}
在sortable创建时触发
start Type: sortstart
当排序动作开始时触发此事件。
sort Type: sort
当元素发生排序时触发此事件。
change Type: sortchange
当元素发生排序且坐标已发生改变时触发此事件。
beforeStop Type: sortbeforestop
在排序动作结束之前触发此事件。此时占位符元素和辅助元素仍有效。
stop Type: sortstop
当排序结束时被触发此事件。
update Type: sortupdate
当排序动作结束时且元素坐标已经发生改变时触发此事件。
receive Type: sortreceive
当一个已连接的sortable对象接收到另一个sortable对象的元素后触发此事件。
remove Type: sortremove
当元素从自己的sortable对象中移出，到了另一个sortable对象中时触发此事件。
over Type: sortover
当元素移动到另一个sortable对象中时触发此事件。
out Type: sortout
当元素从一个连接的sortable对象中移出时触发此事件。
activate Type: sortactivate
当一个有使用连接的sortable对象开始排序动作时，所有相联的sortable都触发此事件。
deactivate Type: sortdeactivate
当一个有使用连接的sortable对象结束排序动作时，所有相联的sortable都触发此事件。
