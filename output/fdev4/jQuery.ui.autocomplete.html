ui-autocomplete : 自动补全

介绍

该组件可以自动筛选、 补全用户输入

版本

  • 倪朱国：创建----2011.10.14

开始

    $.use('ui-autocomplete', function(){
        $('...').autocomplete(options);
    });

Configs

disabled : Bolean
Default : false
使用(false)或禁用(true) autocomplete，可以在初始化autocomplete的时候设置。
appendTo : Selector
Default : "body"
下拉提示菜单应该被追加到哪个元素下
autoFocus : Boolean
Default : false
如果设置为true的第一个选项将被自动聚焦。
delay : Integer
Default : 300
指定从访问者开始在文本域输入到显示下拉提示菜单所间隔的毫秒数；对于本地数据可以将值设为0以加快响应速度。
minLength : Integer
Default : 1
指定用户最少输入多少个字符后启动自动完成组件。对于本地的少量数据可以将值设为0，但有大量的数据时 应该适当的增加这个数值，以免有太多的匹配项。
position : Object
Default : { my: "left top", at: "left bottom", collision: "none"}
设置autocomplete组件基于和他相关的input的相对位置，也可以用"of"选项指定到其他的input上。
source : String, Array, Callback
Default : none, must be specified
数据源，必须指定。

Methods

.autocomplete("destroy")
彻底删除自动补全的功能。这将使input元素返回到初始状态
.autocomplete("disable")
使自动补全功能失效
.autocompelte("enable")
使自动补全功能生效
.autocomplete("option",optionName,[value])
  • optionName : String 参数名
获得或更改autocomplete的参数 
//getter
var optionName = $("...").autocomplete("option",optionName);
//setter
$("...").autocomplete("option",optionName,value);
.autocomplete("option", options)
  • options : Object 参数对象
提供了一个选项obejct,一次设置多个自动补全选项。 
$("...").autocomplete("option",{
    disabled: false,
    appendTo: "body",
    autoFocus: "false",
    delay: 300,
    minLength: 1,
    position: {
        my:"left top",
        at: "left bottom",
        collision:"none"
    },
    source: url
});
.autocomplete("widget")
返回 .ui-autocomplete 元素
.autocomplete("search",[value])
当数据可用时，可以手动触发搜索事件，得到下拉提示菜单。如果没有指定value，将会使用输入框 中的数据。当使用一个空字符和minLenth=0时，可以显示全部项目。
.autocomplete("colse")
关闭打开的菜单。在search方法中很有用，可以关闭已经打开的下拉提示菜单。

Events

create Type: autocompletecreate
  • event :　Event
  • ui : Obejct 
        {
        item: item //当前选中的"item"
    }
autocomplete 创建完成时触发
search Type: autocompletesearch
  • event :　Event
  • ui : Obejct 
        {
        item: item //当前选中的"item"
    }
在请求开始前被触发，可以取消请求(return false)。
open Type: autocompleteopen
  • event :　Event
  • ui : Obejct 
        {
        item: item //当前选中的"item"
    }
在下拉提示菜单打开时被触发。
focus Type: autocompletefocus
  • event :　Event
  • ui : Obejct 
        {
        item: item //当前选中的"item"
    }
当选项得到焦点时（未选中）触发，ui.item为当前选项的引用。它的默认操作为替换输入框中的 值为焦点选项的值，在键盘上下选择选项时被触发。取消该事件可以阻止输入框中的数据被更新，但是不能阻止选项获得焦点。
select Type: autocompleteselect
  • event :　Event
  • ui : Obejct 
        {
        item: item //当前选中的"item"
    }
从下拉提示菜单中选择项目时触发； ui.item是被选中选项的引用。默认动作是用被选中选项的值取代input中的值。 取消此事件，可防止值被取代，但并不妨碍关闭菜单。
close Type: autocompleteclose
  • event :　Event
  • ui : Obejct 
        {
        item: item //当前选中的"item"
    }
当列表隐藏 - 没有发生改变。
change Type: autocompletechange
  • event :　Event
  • ui : Obejct 
        {
        item: item //当前选中的"item"
    }
当字段失去焦点的时候被触发，如果值发生改变， ui.item为被选中选项的引用。
